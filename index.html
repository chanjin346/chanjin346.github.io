01 — Introduction

저를 소개합니다

한화 SW 캠프 백엔드 풀스택 과정을 수료한 신입 개발자입니다. Canon Korea 품질보증팀 경력과 태권도 보조강사 경험을 통해 꼼꼼함과 팀워크를 몸에 익혔습니다.

JWT 인증 시스템 직접 설계부터 CQRS 아키텍처 적용까지, 백엔드 전반을 스스로 설계하고 구현해왔습니다. TDD로 257개의 테스트 케이스를 작성하며, 동작하는 코드보다 검증된 코드를 지향합니다.

부트캠프에서 여러 크고 작은 프로젝트를 경험하면서 기획부터 설계까지 고민하는 과정이 가장 재미있었습니다. 앞으로도 단순히 동작하는 코드가 아닌, 서비스 전체를 이해하고 이유 있는 설계를 할 수 있는 백엔드 개발자로 성장하고 싶습니다.

트러블슈팅

개발 과정에서 맞닥뜨린 기술적 문제와 해결 과정을 기록했습니다.

Security

Feign 역직렬화 실패 — 전역 SNAKE_CASE vs 외부 camelCase

Jackson 전역 설정이 SNAKE_CASE를 강제하면서, 외부 서비스가 반환하는 camelCase JSON 필드가 null로 역직렬화되는 문제. @JsonProperty는 직렬화 방향도 변경해 테스트가 깨짐.

→ @JsonAlias 적용 — 역직렬화만 추가 허용, 직렬화 SNAKE_CASE 유지
Performance

MyBatis N+1 — Cartesian Product 딜레마

ActivityPackage 목록 조회 시 JOIN 방식이 viewer·item 컬렉션 행 수 곱으로 증가하는 Cartesian Product 문제. Feign 단건 반복 호출도 N개 요청 발생.

→ nested select 방식 분리 + Feign 배치 조회 전환
Security

X-User-Id 헤더 신뢰 방식의 보안 취약점

게이트웨이가 주입한 X-User-Id 헤더를 그대로 신뢰하는 구조. 게이트웨이 우회 시 임의 사용자 위조 가능, 마이크로서비스간 직접 호출 시 인증 부재.

→ RS256 JWKS URI 기반 JWT 서명 검증 필터로 완전 전환
Transaction

PDF 생성 @Transactional — 커넥션 풀 점유

PDF 생성 메서드에 @Transactional이 걸려 iText PDF 렌더링 수십 초 동안 DB 커넥션 점유. 동시 요청 시 커넥션 풀 고갈 위험.

→ @Transactional 제거 — DB 조회와 PDF 빌드 단계 명시적 분리
Testing

RS256 전환 후 257개 테스트 전체 실패

JWT 검증 전환으로 @WithMockUser 기반 테스트가 SecurityContext 불일치로 전량 실패. WebMvc 슬라이스와 통합 테스트 설정이 달라 일괄 수정 어려움.

→ JWT mock 설정 통일 + IntegrationTestSupport 공통 베이스 클래스 표준화
Frontend

한글 IME 조합 중 SearchableCombobox 필터링 실패

한글 입력 시 compositionend 이벤트 전까지 v-model 값이 갱신되지 않아 콤보박스 필터링이 동작하지 않는 문제.

→ event.target.value 기반 직접 참조로 IME 조합 중에도 실시간 필터링
